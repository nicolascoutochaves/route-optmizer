Como Funciona

Guia rápido do Otimizador de Roteiros

← Voltar à aplicação

Visão geral

O Otimizador de Roteiros geocodifica endereços e calcula a melhor ordem de visita para cada roteiro entregue. Toda a base de roteiros fica no banco de dados: não é necessário importar ou anexar arquivos — basta entrar com uma conta cadastrada para ter acesso aos roteiros liberados para o usuário.

Sobre o programa

O Otimizador de Roteiros é uma aplicação web front-end (HTML, CSS e JavaScript), hospedada na Vercel. A chave da API de geocodificação (Mapbox) fica protegida em uma função serverless da própria Vercel, nunca exposta no navegador. O banco de dados, a autenticação de usuários e as permissões de acesso são gerenciados pelo Supabase.

O código-fonte é aberto e distribuído sob a licença GNU General Public License v3.0 (GPL-3.0) — consulte o texto completo em gnu.org/licenses/gpl-3.0.

Código aberto não significa dados públicos: qualquer pessoa pode ler ou reaproveitar o código da aplicação, mas os roteiros, endereços e demais dados armazenados no banco continuam privados, protegidos por autenticação e por regras de acesso (RLS) configuradas no Supabase. Só usuários cadastrados e autorizados conseguem ler ou gravar informações.

1 — Login e carregamento automático

Ao abrir a aplicação, clique em 🔐 Entrar e informe e-mail e senha cadastrados. Após a autenticação, os roteiros disponíveis no banco de dados são carregados automaticamente e exibidos na tela inicial — sem necessidade de upload manual de arquivos.

O acesso é restrito a usuários cadastrados. Caso não tenha uma conta, solicite o cadastro ao responsável pelo sistema.

⚠️ Acesso somente leitura: nem todo usuário cadastrado tem permissão de gravação no banco. Se sua conta tiver apenas acesso de leitura, qualquer alteração feita (edição de pontos, reordenação, exclusão) é salva somente no seu navegador (localmente) e some ao limpar os dados do navegador — a aplicação avisa isso na tela sempre que isso acontece. Para que as alterações sejam gravadas de fato no banco e fiquem disponíveis a outros usuários, é necessário ter permissão de escrita.

2 — Seleção do roteiro

Cada roteiro carregado aparece como um cartão na tela inicial, com o nome e a quantidade de pontos. Clique em um cartão para abri-lo na área de processamento e trabalhar com seus pontos.

3 — Geocodificação e otimização

Com o roteiro selecionado, use ⚡ Geocodificar e otimizar para converter os endereços em coordenadas geográficas e calcular a ordem de visita mais eficiente entre os pontos, sempre partindo e retornando ao endereço padrão configurado.

Ao final, a seção de Resultados mostra quantos endereços foram geocodificados com sucesso e quantos não foram encontrados, permitindo correção manual quando necessário.

4 — Edição manual dos pontos

O botão ✏️ Editar roteiro abre um painel onde é possível adicionar, remover ou corrigir nome, endereço e coordenadas de cada ponto, além de travar pontos específicos para que não sejam reordenados pela otimização automática.

Também é possível reordenar os pontos manualmente arrastando-os na lista, para ajustes finos após a otimização automática.

5 — Roteiro personalizado

O painel Roteiro Personalizado permite combinar pontos de diferentes roteiros já carregados em uma nova rota avulsa: escolha o roteiro de origem, selecione os pontos desejados, ajuste a ordem por arraste e gere um link ou QR Code específico para essa combinação.

6 — Link, QR Code e exportação

Depois de gerar a rota, use 🗺️ Google Maps para abri-la diretamente ou 📋 Copiar link para compartilhar o endereço da rota. Um QR Code é gerado automaticamente para abrir a rota em um celular sem precisar digitar nada.

O botão ⬇️ Exportar tudo (KML) gera um arquivo KML com todos os pontos do roteiro, útil para uso em outros aplicativos de mapas ou como backup.

LGPD e privacidade dos dados

Em conformidade com a Lei Geral de Proteção de Dados (Lei nº 13.709/2018), os dados tratados por esta aplicação — e-mail de login, endereços dos roteiros e coordenadas geográficas — são usados exclusivamente para o funcionamento do serviço (autenticação e geocodificação/otimização de rotas).

Esses dados ficam armazenados no banco de dados do Supabase, protegidos por autenticação e por regras de acesso por usuário (Row Level Security), e não são compartilhados com terceiros nem tornados públicos. O fato de o código-fonte ser aberto não dá acesso ao conteúdo do banco de dados.

Para solicitar a exclusão de sua conta ou de dados pessoais associados a ela, entre em contato com o responsável pela administração do sistema.
